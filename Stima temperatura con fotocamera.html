Vuoi stimare la temperatura corporea utilizzando solo la fotocamera del PC, senza dispositivi aggiuntivi, giusto?

Qui ci sono alcuni punti importanti da chiarire:

1. Fotocamera RGB standard ≠ Termocamera

  • Le normali webcam (anche Full HD o 4K) catturano solo luce visibile, non infrarossi.

  • La temperatura corporea è invece legata all’emissione IR (infrarossi, 8–14 μm).

  • Senza un sensore termico (FLIR, termocamera USB, ecc.) non puoi avere una misura diretta.

2. Metodi indiretti con webcam

Se non vuoi/hai una termocamera, puoi solo stimare indirettamente la temperatura corporea usando:

  • Immagini facciali → analisi del flusso sanguigno (remote photoplethysmography, rPPG).

    • Qui si sfruttano microvariazioni del colore della pelle causate dalla circolazione.

    • Puoi stimare battito cardiaco, e in certi casi correlare con stress o febbre.

  • Modelli di deep learning addestrati su dataset che associano caratteristiche visive (come il rossore del volto o la dilatazione dei vasi sanguigni) con la temperatura corporea.

  • OpenPose / MediaPipe Skeleton → puoi tracciare lo scheletro per isolare aree specifiche (fronte, guance, collo), da cui estrarre le feature per la stima.

3. Pipeline possibile con sola webcam

  1. Rileva volto e scheletro (MediaPipe, OpenPose, o OpenCV + dlib).

  2. Isola regioni di interesse (ROI) → fronte, guance.

  3. Estrai segnali rPPG (micro-variazioni nei canali RGB nel tempo).

  4. Feed a un modello ML addestrato su dataset temperatura vs immagine.

  5. Stima la temperatura corporea con un intervallo di incertezza (non sarà precisa come un termometro IR!).

4. Librerie utili

  • MediaPipe → rilevamento volto + scheletro.

  • OpenPose.

  • rPPG Python libs: pyVHR, HeartPy.

  • Modelli ML (TensorFlow / PyTorch) da addestrare su dataset.

Passaggi preliminari

pip install opencv-python dlib numpy

  1. Devi scaricare il file dei pesi di dlib per il riconoscimento dei landmark facciali:
    ➡️ shape_predictor_68_face_landmarks.dat.bz2

  • Decomprimi il file .bz2

  • Metti shape_predictor_68_face_landmarks.dat nella stessa cartella del tuo script Python

Ecco il programma aggiornato:

Rppg Temp Estimation· python
import cv2
import dlib
import numpy as np
from collections import deque

# Carica modello landmark facciali (68 punti)
predictor_path = "shape_predictor_68_face_landmarks.dat"
detector = dlib.get_frontal_face_detector()
predictor = dlib.shape_predictor(predictor_path)

# Inizializzazione OpenCV webcam
cap = cv2.VideoCapture(0)

# Buffer per il segnale rPPG
signal_buffer = deque(maxlen=250) # circa 10 secondi a 25 fps

while True:
ret, frame = cap.read()
if not ret:
break

gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
faces = detector(gray, 0)

for face in faces:
landmarks = predictor(gray, face)

# Prendiamo alcuni punti della fronte (circa tra le sopracciglia e metà fronte)
forehead_points = [
(landmarks.part(19).x, landmarks.part(19).y - 40), # sopra sopracciglio sinistro
(landmarks.part(24).x, landmarks.part(24).y - 40), # sopra sopracciglio destro
(landmarks.part(24).x, landmarks.part(24).y - 10),
(landmarks.part(19).x, landmarks.part(19).y - 10)
]

forehead_coords = np.array(forehead_points, dtype=np.int32)

# Maschera ROI
mask = np.zeros(frame.shape[:2], dtype=np.uint8)
cv2.fillPoly(mask, [forehead_coords], 255)
roi = cv2.bitwise_and(frame, frame, mask=mask)

# Calcolo media canale verde (rPPG)
green_values = roi[:, :, 1][mask == 255]
if green_values.size > 0:
mean_green = np.mean(green_values)
signal_buffer.append(mean_green)

# Disegno rettangolo fronte
cv2.polylines(frame, [forehead_coords], True, (0, 255, 0), 2)

# Mostra frame
cv2.imshow('Webcam + Forehead ROI (dlib)', frame)

# Uscita con ESC
if cv2.waitKey(1) & 0xFF == 27:
break

cap.release()
cv2.destroyAllWindows()
